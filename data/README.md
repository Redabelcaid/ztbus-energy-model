# `data/`

This tree is **not** tracked by git. It holds large generated artifacts;
the schema and contents are regenerated by the pipeline.

```
data/
├── raw/         # original ZTBus CSVs (read-only). On HPC, symlink to scratch.
├── interim/     # raw → Parquet, schema-validated, mission-partitioned
├── processed/   # cleaned + feature-engineered, mission-partitioned
└── reports/     # QC summaries, plots, fit diagnostics
```

## Getting the raw data

Download from the ETH Research Collection:
<https://www.research-collection.ethz.ch/entities/researchdata/61ac2f6e-2ca9-4229-8242-aed3b0c0d47c>

You should end up with ~1409 files named `B{bus}_{date}_{time}_{date}_{time}.csv`
plus a `metaData.csv`. Place them under `data/raw/` (or symlink the directory).

## On HPC

Use scratch / project storage for everything under `data/`. The repo's
`data/raw` should be a symlink to wherever the cluster lets you put 10 GB of
small files. See `docs/hpc_runbook.md`.
